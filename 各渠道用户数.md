### 统计每天各渠道独立访客数的链路

- [ ] **Kafka → Flink 接收与过滤**

  Flink 消费用户行为日志，首先过滤掉没有 userId 的匿名请求，然后提取统计维度（渠道、页面等）和时间戳，按照统计粒度归属到对应的时间窗口（比如 14:30 这一分钟）

- [ ] **Flink → Redis Bitmap 判断新老访客**

  Flink 用 **Async IO** 异步请求 Redis，执行SETBIT 返回该位的旧值，**返回 0 = 新访客，返回 1 = 老访客**。只有返回 0 的数据才进入后续计数，返回 1 的直接丢弃。

- [ ] **Flink 窗口聚合**

  新访客事件进入 Tumbling Window，窗口时长就是你的统计粒度（1 分钟或 1 小时）。窗口触发时，把这段时间内的新访客数 SUM 起来，输出一条聚合结果

- [ ] **写入 Doris**

  用 Unique Key 模型，`channel + window_start` 作为 Key，uv 作为 Value。如果有迟到数据触发窗口重算，Doris 的 Upsert 会直接更新那一行，看板拿到的始终是最新值
